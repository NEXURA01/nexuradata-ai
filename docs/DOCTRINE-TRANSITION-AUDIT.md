# NEXURADATA Doctrine Transition Audit

## Purpose

This audit maps the repository against the new doctrine: operational intelligence infrastructure, workflow orchestration, execution systems, operational clarity, and AI-assisted operational environments.

The current risk is structural and narrative, not only technical. The platform has been repositioned, but the repository still contains older service language, legacy architecture, and visual patterns that can dilute perception.

## Risk Statement

The main current risk is no longer purely technical. It is structural and narrative.

NEXURADATA has moved toward operational intelligence infrastructure centered on:

- workflow systems
- orchestration
- execution infrastructure
- operational clarity
- AI-assisted operational environments

The repository still contains:

- old marketing language
- references to the previous business model
- inherited visual patterns
- service-oriented pages
- generic or startup-style formulations
- Neon and older architecture traces
- components that do not fully reflect the current doctrine

The major risk becomes:

- perception incoherence
- narrative fragmentation
- diluted positioning
- conflict between old and new identity
- non-unified user experience

## Classification Model

| Status | Meaning | Action |
| --- | --- | --- |
| KEEP | Already aligned with the new doctrine | Preserve and refine only when needed |
| REFRAME | Useful structure, wrong language or UX | Rewrite/reposition without deleting value |
| LEGACY | Old positioning still active | Keep temporarily, route into transition plan |
| REMOVE LATER | Obsolete or contradictory | Remove only after replacement path exists |

## Design Audit Targets

Search and classify:

- blue/purple color values
- large border radii
- random gradients

- generic cards
- decorative icons
- noisy motion
- duplicated layout patterns

Visual corrections should reduce, sharpen, darken, align, or remove.

## Priority Sequence

1. Protect current working behavior.
2. Use `docs/DOCTRINE.md` as the source of truth.
3. Keep homepage and pricing aligned as current doctrine anchors.
4. Reframe high-SEO legacy pages gradually.
5. Preserve Neon routes until Supabase equivalents are tested.
6. Turn status/intake/operator surfaces into the real operational engine.
7. Remove legacy pages only after replacement routes and SEO strategy exist.
