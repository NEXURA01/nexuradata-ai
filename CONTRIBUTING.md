# Contributing to NEXURA

Thank you for taking the time to improve this repository. This project supports the public NEXURA operational intelligence infrastructure site.

## Before You Start

- Open an issue for meaningful behavior, security, content, or deployment changes before submitting a pull request.
- Keep changes focused and small enough to review clearly.
- Do not add private addresses, personal contact details, credentials, client data, or local machine paths.
- Use business contact language only for public-facing content.

## Development Workflow

1. Create a branch from `main`.
2. Make the smallest change that solves the issue.
3. Run the local checks:

```bash
npm run check
npm test
npm run build
```

4. Open a pull request using the repository template.

## Content And Design Guidelines

- Keep the positioning premium, operational, calm, and precise.
- Prefer structure, clarity, trust, and execution language over generic AI startup copy.
- Do not introduce personal identity references, private addresses, or unrelated brand names.
- Keep visual changes consistent with the existing NEXURA design system.

## Security

Do not report vulnerabilities through public issues. Use the process in [SECURITY.md](SECURITY.md).
